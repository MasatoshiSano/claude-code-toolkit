# 🔧 Commit - 従来のコミット形式で自動コミット

## 概要
変更内容を分析し、従来のコミット形式（Conventional Commits）で適切なコミットメッセージを作成してコミットを実行します。

**コミット完了後、技術ブログ記事作成を自動提案します。**

## 実行手順

1. **変更内容の確認**
   - `git status` で変更ファイルを確認
   - `git diff` で具体的な変更内容を分析

2. **コミットメッセージの生成**
   - 変更の性質を分析（feat/fix/docs/style/refactor/test/chore）
   - 適切な絵文字を選択
   - 簡潔で明確なメッセージを作成

3. **コミット実行**
   - ステージングされていないファイルを自動で `git add`
   - 生成されたメッセージでコミット実行
   - コミット結果を確認

4. **技術ブログ記事作成の提案（新機能）**
   - コミット完了後、変更内容を分析
   - 技術ブログにふさわしい内容を検出
   - ユーザーに記事作成を提案

## コミット形式
```
<type>(<scope>): <subject> <emoji>

<body>
```

例：
- `feat: ユーザー認証機能を追加 ✨`
- `fix: ログイン時のエラーハンドリングを修正 🐛`
- `docs: APIドキュメントを更新 📝`

## ブログ記事作成提案の例

コミット完了後：
```
✓ コミット完了: feat: useCallbackでパフォーマンス改善 ✨

📝 技術ブログの作成候補を検出しました：
1. useCallbackで関数の再生成を防ぐ方法（推奨度: 高）
   - 対象読者: React初心者
   - 推定文字数: 1500-2000字

作成しますか？ [y/N]
```

「y」を選択すると `/blog` コマンドが自動実行され、対話形式で記事を作成します。

## 使用方法
```
/commit
```

変更内容に基づいて自動的に最適なコミットメッセージを生成し、コミットを実行します。
コミット完了後、技術ブログにふさわしい内容があれば記事作成を提案します。
