description: "全変更をaddし、Qiita記事を作成してコミットする"
---

現在の作業ディレクトリにある変更内容(まだステージングしていない変更も含む)をすべて分析し、以下の手順を実行してください。

## Step 1: 変更内容の把握と記事執筆

1. `git diff HEAD` などのコマンドを用いて、前回のコミットからの変更点をすべて読み取ってください。
2. その内容に基づき、QiitaのFrontmatter形式を含むMarkdownファイルを作成してください。
   - ファイルパス: `docs/qiita/YYYYMMDD_topic.md` (日付と適切なトピック名を使用)
   - **記事フォーマット**:
     ```markdown
     ---
     title: {変更内容を反映したタイトル}
     tags: {技術タグ 3~5個}
     private: true
     updated_at: 'YYYY-MM-DD'
     ---
     ## 実装の背景
     {変更を行った理由や課題を記述}

     ## やったこと
     {具体的な変更内容を箇条書きまたは詳細に記述}

     ## 実装の詳細
     {コード例や重要な変更箇所の説明}

     ## まとめ
     {今回の実装の効果や今後の展望}
     ```

## Step 2: 一括コミット

1. **`git add .` を実行**し、すべての変更(コード修正 + 新規作成した記事)をステージングしてください。
2. 変更内容を要約したメッセージで `git commit` を実行してください。
   - コミットメッセージは変更内容を適切に要約したものにしてください。

## 注意事項

- Qiita記事のファイル名は `YYYYMMDD_topic.md` の形式にしてください(例: `20251201_add_user_auth.md`)
- 記事のタイトルは変更内容を適切に表現してください
- タグは関連する技術スタック(例: Python, FastAPI, Docker, AWS など)を3~5個選んでください
- `private: true` を設定し、公開前にレビューできるようにしてください
