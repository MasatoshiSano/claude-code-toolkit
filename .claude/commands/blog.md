# /blog

## 概要

開発者の実装内容に基づき、**対話形式で**技術ブログ記事の**ドラフト（たたき台）**を作成するアシスタントです。完全自動生成ではなく、開発者との共同作業により、質の高い記事作成の初動をサポートします。

## 実行フロー

1.  **範囲の指定 (Scope Definition)**
    - ユーザーは引数で記事化したい実装の範囲をエージェントに伝えます。
    - 例: 現在のブランチ、特定のコミット範囲、ステージング中の変更など。

2.  **履歴の分析 (Analysis)**
    - 指定された範囲の `git` コミット履歴、関連するソースコードの差分を分析します。
    - `_docs/` ディレクトリ内の関連ドキュメントやログを読み込みます。
    - ソースコード内の特殊コメント（例: `// BLOG: ここが今回のパフォーマンス改善の核心`）を優先的に抽出し、文脈を理解するヒントにします。

3.  **対話によるヒアリング (Interactive Q&A)**
    - 分析結果を基に、エージェントがユーザーに質問を投げかけ、記事の方向性や背景を深掘りします。
    > **質問例:**
    > - 「分析の結果、テーマは『非同期処理のパフォーマンス改善』でよろしいでしょうか？」
    > - 「この記事の主な読者層（ターゲット）は誰を想定しますか？」
    > - 「`ApiService` の変更について、特に苦労した点や工夫した点は何ですか？」
    > - 「タイトル案を3つ提案します。どれがイメージに近いですか？」

4.  **ドラフト記事の生成 (Draft Generation)**
    - ヒアリング内容を基に、Markdown形式でブログ記事のドラフトを生成します。
    - これは完成品ではなく、あくまでユーザーが加筆・修正するための**骨子**です。

5.  **保存 (Save)**
    - 生成された記事を `_docs/blog/` ディレクトリに `DRAFT-` プレフィックスを付けて保存し、ユーザーがすぐに編集を開始できるようにします。

## 記事テンプレート

-   **タイトル**: 対話を通じて、魅力的で内容を的確に表すタイトルを生成
-   **概要**: 何を解決し、何を学んだかを簡潔に記述
-   **課題と背景 (Why)**: なぜこの実装が必要だったのか、解決したかった課題は何か
-   **技術スタック**: 使用した技術・ツール
-   **実装内容**: 具体的な実装方法と、対話で選択された本質的なコード例
-   **学んだこと**: 試行錯誤の過程、失敗や改善点も含めた深い学び
-   **まとめ**: 他の開発者への応用可能性や今後の展望

## 対象内容

-   React/Next.js の開発
-   Python, Djangoでのwebアプリ開発
-   Dockerにおけるweb環境開発
-   テスト駆動開発（TDD）の実践
-   パフォーマンス最適化
-   ユーザー体験の改善
-   開発ツールの活用

## 出力例

```

\_docs/blog/DRAFT-2025-09-27\_Jestテスト環境の構築と包括的テストスイート.md

```

## 使用例

#### 例1: 現在のブランチの差分を記事にする場合

```

/blog current-branch

```
> **Agent:** 現在のブランチ (`feature/user-auth`) の差分を分析しました。テーマは「JWTを用いた認証機能の実装」でよろしいでしょうか？
>
> **User:** はい、それでお願いします。
>
> **Agent:** 承知しました。この記事で特に技術的に強調したいポイントはどこですか？

#### 例2: 特定のコミット範囲を記事にする場合

```

/blog 2b882dd..e1b74e1

```
> **Agent:** 指定されたコミット範囲を分析しました。`useState` から `useReducer` へのリファクタリングが中心のようですね。この変更を行った一番の理由は何ですか？
>
> **User:** 複雑な状態管理をシンプルにしたかったからです。

## 主な改善点

1.  **対話による精度の向上**
    単なる自動生成ではなく、ユーザーとの対話を通じて「なぜ」という背景や意図を汲み取り、記事の質を大幅に向上させます。

2.  **ユーザーによるスコープ制御**
    ユーザーが引数で範囲を明確に指定することで、「何について書くか」をコントロールでき、意図しない内容の記事が作られるのを防ぎます。

3.  **「アシスタント」としての位置づけ**
    出力が「ドラフト」であることを明確にし、ユーザーの期待値を適切に管理します。これにより、ツールは「完璧な代筆者」ではなく「思考を整理し、執筆を加速させるパートナー」となります。
